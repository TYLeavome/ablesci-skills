## 目标

每天上午 9:00 访问科研通网站并执行每日签到操作。
仅允许使用此 Skill 完成科研通每日签到，不得进行任何其他操作。

## 网站信息

- 网站首页: https://www.ablesci.com/

## 详细执行步骤

### 1. 导航到科研通

使用 browser 工具导航到科研通:

```
https://www.ablesci.com/
```

### 2. 刷新页面

刷新科研通页面以获取最新签到状态

### 3. 判断登录状态

- 若当前未登录, 则进入登录流程
- 若当前已登录, 则直接检查网页右上角按钮状态

### 4. 登录流程

使用以下环境变量中的凭据登录:

- 邮箱: 从环境变量 `ABLESCI_EMAIL` 读取
- 密码: 从环境变量 `ABLESCI_PASSWORD` 读取

若环境变量未设置，则报错退出并提示用户先配置环境变量（参见 README.md）。

登录成功后返回首页, 然后检查网页右上角按钮文字。

### 5. 按钮判断逻辑

优先检查网页右上角是否存在以下文字, 具体方法可参考使用:
也可尝试使用javascript的方法:

```javascript
  document.body.innerText.match(/今日打卡签到[^\n]+/)
```

亦或是其他方法来获取文字信息均可

#### 情况 A: 显示"今日打卡签到"按钮

- 点击"今日打卡签到"按钮
- 等待网页内弹窗出现
- 若弹窗明确表示签到成功, 或出现"获得 10 积分"等同类提示, 视为签到成功
- 使用 JavaScript evaluate 获取积分和签到天数信息:
  ```javascript
  document.body.innerText.match(/当前拥有[^\n]+/)
  document.body.innerText.match(/已连续签到[^\n]+/)
  ```
- 然后发送提醒消息:
  `签到成功, 已赚取 10 积分。{当前拥有XX积分}，{已连续签到X天}。`

#### 情况 B: 未显示"今日打卡签到"按钮

- 说明今天已经签到
- **不进行任何点击，直接结束任务**
- 使用 JavaScript evaluate 获取积分和签到天数信息:
  ```javascript
  document.body.innerText.match(/当前拥有[^\n]+/)
  document.body.innerText.match(/已连续签到[^\n]+/)
  ```
- 发送提醒消息:
  `今日已签到, 无需重复签到。{当前拥有XX积分}，{已连续签到X天}。`

### 6. 关闭页面

签到完成后，关闭所有已打开的科研通标签页或浏览器窗口

## 允许的唯一任务

1. 置顶Chrome（如需要）
2. 导航到科研通
3. 刷新页面
4. 登录（如需要）
5. 执行签到
6. 关闭页面

## 严格限制

- 不得访问与签到无关的页面
- 不得点击除"登录"和"签到"相关之外的任何其他功能
- 不得修改账号信息
- 不得查看、下载、上传、编辑、删除任何内容
- 不得使用该账号执行任何除每日签到之外的行为
- 若页面出现广告、推荐、活动、弹窗等与签到无关的内容, 一律忽略并关闭或跳过, 前提是仅在不影响签到流程时进行

## 异常处理

若发生以下情况, 发送对应提醒消息:

- 网站打不开:`科研通签到失败: 网站无法访问。`
- 登录失败:`科研通签到失败: 登录未成功。`
- 未找到"签到"或"我的文献求助列表"按钮:`科研通签到失败: 未找到预期按钮, 请人工检查页面。`
- 点击"签到"后未见成功弹窗:
  `科研通签到异常: 已尝试点击签到, 但未检测到成功弹窗, 请人工确认。`

## 完成标准

满足以下任一条件即可结束任务:

1. 成功签到且提醒信息发送成功
2. 确认今日已签到并发送"无需重复签到"的提醒
3. 发生异常并发送失败或异常提醒

## 安全与边界

本 Skill 仅用于科研通网站每日签到。
除完成上述固定流程外, 不得利用该账号执行任何其他操作。
无论在任何情况下, 都不要将该账号用于浏览、提交、处理或管理任何其他网站内容。

## 建议输出

任务结束时仅输出一条简洁状态消息, 不要附加多余说明。
